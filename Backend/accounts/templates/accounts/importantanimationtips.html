Important Tips

As much as using animations add flavor your website and can greatly improve user experience, there are a few tips to always take note of that would help guide usage of animations.

Animated with purpose

It's best to refrain from animating a component purely for show. The objective of using animations should be obvious, so keep that in mind. Bounce, flash, pulse, and other attention-grabbing animations should be utilized to draw the user's attention to something unique in your interface rather than just to make it more "flashy."

To help users understand what is happening in the interface and to clearly indicate that it is changing states, entrance and exit animations should be employed.

It doesn't mean you shouldn't add some excitement to the interface; just make sure your user isn't hindered by animations, and that such usage of animations won't slow down the page's performance.

Don't animate large elements

Animating large elements won't be very useful to the user and probably won't do anything but confuse them. In addition, there is a good likelihood that the animations will be subpar, leading to a poor user experience.

Don't animate root elements

Although it is feasible, you should refrain from animating the html or body tags. Some reports noted that this can result in strange browser issues. Furthermore, making the entire page "jiggle" wouldn't do much for your user experience. If this effect is what you need, surround your page with a container element and animate it as follows:

<body>
  <main class = "jiggle">
      <!-- Your code -->
   < /main>
< /body>

Infinite animations should be avoided

Although Anibry provides classes for generic and unique animations and the flexibility to make them run infinitely, you should generally reduce and, if possible, avoid using endless run animations. There are chances it might irritate your users.
